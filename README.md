# 发票提交系统

一个简单的发票数据采集和管理系统，支持员工提交发票信息和管理员查看、导出数据。

## 功能特点

### 表单端 (index.html)
- 个人信息填写：姓名、部门、发票金额、提交时间
- 发票附件上传：支持JPG、PNG、PDF格式
- 表单验证：实时验证输入数据的合法性
- 提交状态反馈：提交成功后显示确认信息
- 响应式设计：适配移动端和桌面端

### 管理端 (admin.html)
- 数据可视化：部门分布图表、金额趋势图表
- 数据筛选和搜索：按姓名、部门搜索筛选
- 数据排序：支持多字段排序
- 数据导出：支持CSV和Excel格式导出
- 批量操作：批量导出、批量删除
- 统计信息：总提交数、总金额、部门数等统计

## 技术栈

- HTML5
- CSS3 (Tailwind CSS v3)
- JavaScript (原生)
- Font Awesome 图标
- Chart.js 图表库

## 本地存储

本系统使用浏览器的localStorage存储数据，适用于小型团队或测试环境。如需更可靠的数据存储，建议后端集成。

## 部署说明

### Vercel 部署

1. 登录 Vercel 账号
2. 连接 GitHub 仓库
3. 选择要部署的仓库
4. 配置部署设置：
   - 框架预设：静态网站
   - 构建命令：无需构建
   - 输出目录：根目录
5. 点击部署

### GitHub Pages 部署

1. 进入 GitHub 仓库设置
2. 找到 Pages 选项
3. 选择分支：main 或 master
4. 选择文件夹：/(根目录)
5. 点击保存，等待部署完成

## 使用说明

1. 将项目部署到网络服务器
2. 访问 index.html 查看表单页面
3. 访问 admin.html 查看管理页面（实际应用中应添加访问控制）
4. 员工可通过扫描二维码访问表单页面提交发票信息
5. 管理员可在管理页面查看、筛选、导出数据

## 数据安全

请注意，本系统使用前端本地存储，不包含后端验证和安全措施。在生产环境中，建议：

1. 添加用户认证系统
2. 使用后端数据库存储数据
3. 实现文件上传到服务器
4. 添加数据备份机制
